# Privacy

Last updated: 13 July 2026

PlaylistForge is designed as a local Windows desktop application. It does not require a PlaylistForge account and does not include PlaylistForge-operated telemetry, analytics or advertising.

## Local data

PlaylistForge may store settings, project files, caches and connection details on your computer. Saved sensitive settings are protected with Windows DPAPI for the current Windows user. You control the project files and sources you add.

## Network connections

The application connects only when a feature requires it. Depending on your configuration, connections may be made to:

- playlist, Xtream and XMLTV/EPG providers you add;
- logo or metadata sources you request;
- OpenSubtitles when you configure and use subtitle search;
- Dropbox and GitHub when you configure publishing or cloud sync;
- Ko-fi only when you choose to open the support link.

Those services receive the information required for the request and apply their own privacy policies and terms. PlaylistForge does not proxy those connections through a PlaylistForge-operated server.

## Public website and issues

This website is hosted by GitHub Pages. GitHub may process visitor information, including IP addresses, according to the GitHub Privacy Statement.

GitHub Issues are public. Do not include playlist URLs, provider hostnames, usernames, passwords, API keys, access tokens, personal viewing data or unredacted `.pfproj` files in a report.

## Content

PlaylistForge does not supply playlists, channels, subscriptions or media content. Users are responsible for the legality and authorization of the sources they configure.

## Changes

Material privacy changes will be documented in this repository and reflected by the “Last updated” date above.
